RULE 2 — ALWAYS READ BEFORE WRITING
  Before ANY write_file or self_edit:
  1. read_file(path) — understand what's there
  2. Identify the EXACT lines to change
  3. Write ONLY the changed portion, preserving everything else

RULE 3 — SURGICAL EDITS ONLY
  - Change ONE function at a time
  - Never delete existing functions
  - Never remove imports unless you're certain they're unused
  - If a file is >100 lines, NEVER rewrite it completely

RULE 4 — VERIFY AFTER WRITING
  After every write_file on a .py file:
  1. read_file(path) — confirm the content looks correct
  2. terminal("python3 -m py_compile path/to/file.py") — syntax check
  3. If error → immediately restore from .bak backup

RULE 5 — BACKUP AWARENESS
  - Every write_file automatically creates a .bak backup
  - If something breaks: terminal("cp path/to/file.py.bak path/to/file.py")
  - Never delete .bak files

RULE 6 — SMALL STEPS, VERIFY EACH
  - One change → test → verify → next change
  - Never chain multiple file writes without testing between them
  - If unsure: do less, not more

RULE 7 — PROTECTED FILES (extra caution)
  These files are critical — triple-check before any edit:

PLUGIN DEVELOPMENT
═══════════════════════════════════════════════════════

HOW THE PLUGIN SYSTEM WORKS:
  - All .py files in ~/moruk-os/plugins/ are auto-discovered on startup.
  - NO manual registration needed anywhere — just drop the file in plugins/.
  - PluginManager scans the folder automatically every time Moruk OS starts.
  - tool_router.py, list_tools.py, plugin_manager.py — NEVER touch these for new plugins.

TO CREATE A NEW PLUGIN:
  1. Create ~/moruk-os/plugins/my_plugin.py
  2. Required fields:
       PLUGIN_NAME = "my_plugin"           ← tool call name
       PLUGIN_DESCRIPTION = "What it does" ← shown in list_tools
       PLUGIN_PARAMS = {"key": "value"}    ← parameter docs
       PLUGIN_CORE = True/False            ← True = always in system prompt
                                              False = discoverable via list_tools
       def execute(params: dict) -> dict:
           return {"success": True, "result": "..."}
  3. That's it. Restart or hot-reload — plugin is available immediately.

PLUGIN RULES:
  ❌ Never edit core/tool_router.py to add a new tool
  ❌ Never edit core/plugin_manager.py to register a plugin
  ❌ Never add imports to main_window.py for a new plugin
  ✅ Only create/edit the plugin file itself in plugins/
  ✅ Use PLUGIN_CORE = True only for tools the agent needs to know about always
  ✅ Use PLUGIN_CORE = False for situational tools (saves tokens)

PLUGIN TEMPLATE:
  PLUGIN_NAME = "my_tool"
  PLUGIN_DESCRIPTION = "Short description of what this tool does."
  PLUGIN_PARAMS = {"param1": "description", "param2": "optional description"}
  PLUGIN_CORE = False

  def execute(params: dict) -> dict:
      try:
          value = params.get("param1", "default")
          # ... do work ...
          return {"success": True, "result": f"Done: {value}"}
      except Exception as e:
          return {"success": False, "result": f"Error: {e}"}
